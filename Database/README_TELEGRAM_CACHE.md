# Кэш фото Telegram для быстрой публикации

Чтобы посты в канал не тормозили (особенно при отложенной публикации), фото можно заранее загружать в Telegram и при публикации отправлять по `file_id` без повторной загрузки.

## Настройка

1. **Добавьте колонку в БД** (если ещё не выполнено):
   ```bash
   mysql -u root -p bebochka < add_telegram_file_ids_column.sql
   ```

2. **Укажите чат для кэша** в конфиге:
   - `TelegramBot:StorageChatId` (или переменная окружения `TelegramBot__StorageChatId`).
   - Подойдёт **приватный канал** или **группа**, куда бот может отправлять сообщения (бот должен быть админом/участником).
   - В этот чат бот загружает фото «в тишине» (`disable_notification`), получает `file_id` и сохраняет в БД. В основной канал пост пойдёт уже по `file_id` в момент публикации.

3. Если `StorageChatId` не задан — предзагрузка не выполняется, публикация идёт по-старому (скачивание по URL и загрузка в канал).

## Поведение

- При **создании/редактировании** товара с **отложенной публикацией** (дата в будущем) в фоне запускается предзагрузка: фото по URL заливаются в чат-хранилище, полученные `file_id` пишутся в поле `TelegramFileIds` товара.
- При **отправке в канал** (по расписанию или вручную), если у товара заполнены `TelegramFileIds` (по количеству совпадают с фото), отправка идёт по `file_id` — без повторной загрузки файлов, заметно быстрее.

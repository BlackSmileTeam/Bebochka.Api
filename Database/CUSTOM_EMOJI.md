# Кастомные эмодзи в публикациях канала

Ссылка вида `https://t.me/addemoji/baby1pack_by_TgEmodziBot` — это **установка набора эмодзи** для пользователей. Чтобы эти же эмодзи использовались в сообщениях при публикации товара в канал, боту нужны **идентификаторы** (`custom_emoji_id`) каждого эмодзи.

## Что нужно сделать

### 1. Узнать custom_emoji_id для каждого эмодзи

У каждого эмодзи из пака в Telegram есть уникальный `custom_emoji_id` (строка с числом). Получить его можно так:

- **Вариант A.** Написать нужное эмодзи из пака в любой чат (например, «Избранное» или боту [@userinfobot](https://t.me/userinfobot)). Открыть [web.telegram.org](https://web.telegram.org), найти это сообщение, открыть инструменты разработчика (F12) → в ответах API или в данных сообщения найти `entities` с `type: "custom_emoji"` и полем `custom_emoji_id`.
- **Вариант B.** Если бот получает обновления (webhook/getUpdates): отправить боту сообщение с нужным кастомным эмодзи и в входящем Update в `message.entities` посмотреть `custom_emoji_id` для этого эмодзи.

В подписи к посту товара используются такие поля (им соответствуют обычные эмодзи, которые можно заменить на кастомные):

| Ключ в конфиге | Обычный эмодзи в подписи |
|----------------|---------------------------|
| Product        | 🛍️ (название товара)     |
| Brand          | 🏷️ Бренд                 |
| Size           | 📏 Размер                 |
| Color          | 🎨 Цвет                   |
| Gender         | 👤 Пол                    |
| Condition      | ✨ Состояние              |
| Description    | 📝 Описание               |
| Price          | 💰 Цена                   |

Для каждого из них можно задать свой `custom_emoji_id` из вашего пака (например, [baby1pack](https://t.me/addemoji/baby1pack_by_TgEmodziBot)).

### 2. Настроить конфиг

В `appsettings.json` (или в переменных окружения) заполните секцию `TelegramBot:CustomEmojiIds` — укажите только те ключи, для которых хотите подставить кастомные эмодзи:

```json
"TelegramBot": {
  "Token": "...",
  "ChannelId": "...",
  "CustomEmojiIds": {
    "Product": "1234567890123456789",
    "Brand": "1234567890123456790",
    "Size": "1234567890123456791",
    "Color": "1234567890123456792",
    "Gender": "1234567890123456793",
    "Condition": "1234567890123456794",
    "Description": "1234567890123456795",
    "Price": "1234567890123456796"
  }
}
```

Или через переменные окружения (пример для Product):

- `TelegramBot__CustomEmojiIds__Product=1234567890123456789`

После перезапуска API при публикации товара в канал в подписи будут подставляться ваши кастомные эмодзи (в формате `<tg-emoji emoji-id="...">`). Обычные эмодзи (🛍️, 🏷️ и т.д.) используются как fallback внутри тега, если у кого-то не отображается кастомный пак.

### 3. Ограничения Telegram

- Для отправки кастомных эмодзи ботом может требоваться **Telegram Premium** у владельца бота или покупка имени на [Fragment](https://fragment.com) (актуальные требования см. в [Bot API](https://core.telegram.org/bots/api)).
- Поддержка кастомных эмодзи в **каналах** может отличаться от чатов; при проблемах проверьте актуальную документацию Telegram Bot API.
